Hoverbot

Hoverbot: Using jQuery for minigames

Client: Personal Project
Role: UX/UI, Coding & Game Design
Tools Used: HTML5, CSS3, jQuery, Photoshop & Illustrator
Duration: 2 weeks



OVERVIEW


Hoverbot is a minigame where you control a robot with the objective of collecting the highest number of orbs and survive the laser beams. The game was developed with a combination of HTML5, CSS3 & jQuery. My goal with this project was to learn more about the possibilities of jQuery in creating interactive elements and animations.




Creating a challenging and fun gaming experience

Hoverbot Level 1

The following information is about the development process of this project, including each step from concept to the final design and challenges that I faced during the development.


CONCEPT & DESIGN


jQuery is an effective tool to create animations and interactions. Since my objective was to create a minigame using this JavaScript library, I began sketching some ideas that I believed were appropriated for this project. My decision was to make a 2D minigame with some puzzles and a time limit to add some challenge.

The Concept

After the creation of a few concepts, I initiated the coding of a demo to visualize the primary game mechanics and general elements. I implemented a timer, animations and included some objects to represent the obstacles & items.

Animations

At this point, I was satisfied with the concept. I started to code the necessary functionalities of the game and since many of the scripts had a "hover" event my decision was to name it Hoverbot.


With the implementation of the mechanics, the next step was to work on the visual aspect of the minigame. I was interested to create something with a futuristic style & a space setting. The background would be different views of the universe and the obstacles would be laser beams. For the main characters, I designed 3 robots that the player can choose before starting the game.

RESULT


The art elements of the game were finalized and all issues that I encountered during the testing phase were fixed. Another goal of this project was to finish it in a short period of time. All assets were created in less than 2 weeks and the final result was a minigame with 4 levels and puzzles.
